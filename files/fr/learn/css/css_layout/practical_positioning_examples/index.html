Objectif: Avoir une idée des aspects pratiques du positionnement

Une boîte d'information à onglets

Le premier exemple que nous allons examiner est une boîte d'information à onglets classique - une fonction très courante utilisée lorsque vous voulez regrouper beaucoup d'informations dans une petite zone. Cela inclut les applications gourmandes en informations comme les jeux de stratégie/guerre, les versions mobiles de sites Web où l'écran est étroit et l'espace limité, et les boîtes d'information compactes où vous pourriez vouloir rendre beaucoup d'informations disponibles sans qu'elles remplissent toute l'interface utilisateur. Notre exemple simple ressemblera à ceci une fois que nous aurons terminé :

Note: Vous pouvez voir l’exemple terminé en direct live à info-box.html (source code). Consultez-le pour avoir une idée de ce que vous allez construire dans cette section de l’article.

Vous pourriez vous dire "pourquoi ne pas créer des onglets séparés sous forme de pages web séparées, et faire en sorte que les onglets permettent de cliquer sur les pages séparées pour créer l'effet ?" Ce code serait plus simple, oui, mais alors chaque "page" séparée serait en fait une nouvelle page web, ce qui rendrait plus difficile la sauvegarde des informations entre les vues, et intégrerait cette fonctionnalité dans un design d'interface plus large. De plus, les applications dites " à page unique " deviennent très populaires - en particulier pour les interfaces Web mobiles - parce que le fait que tout soit servi dans un seul fichier réduit le nombre de requêtes HTTP nécessaires pour voir tout le contenu, ce qui améliore les performances.

Note: Certains développeurs Web amènent les choses encore plus loin, n’ayant qu’une seule page d’informations chargée à la fois et modifiant dynamiquement les informations affichées à l’aide d’une fonctionnalité JavaScript telle que XMLHttpRequest. À ce stade de votre apprentissage, cependant, nous voulons garder les choses aussi simples que possible. Il y a un peu de JavaScript plus tard, mais seulement un tout petit peu.

Pour commencer, nous aimerions que vous fassiez une copie locale du fichier HTML de départ — info-box-start.html. Enregistrez ce fichier dans un endroit approprié sur votre ordinateur et ouvrez-le dans votre éditeur de texte. Examinons le HTML contenu dans le body:

<section class="info-box">
  <ul>
    <li><a href="#" class="active">Tab 1</a></li>
    <li><a href="#">Tab 2</a></li>
    <li><a href="#">Tab 3</a></li>
  </ul>
  <div class="panels">
    <article class="active-panel">
      <h2>The first tab</h2>

      <p>Lorem ipsum dolor sit amet, consectetur adipiscing elit. Pellentesque turpis nibh, porttitor nec venenatis eu, pulvinar in augue. Vestibulum et orci scelerisque, vulputate tellus quis, lobortis dui. Vivamus varius libero at ipsum mattis efficitur ut nec nisl. Nullam eget tincidunt metus. Donec ultrices, urna maximus consequat aliquet, dui neque eleifend lorem, a auctor libero turpis at sem. Aliquam ut porttitor urna. Nulla facilisi.</p>
    </article>
    <article>
      <h2>The second tab</h2>

      <p>This tab hasn't got any Lorem Ipsum in it. But the content isn't very exciting all the same.</p>
    </article>
    <article>
      <h2>The third tab</h2>

      <p>Lorem ipsum dolor sit amet, consectetur adipiscing elit. Pellentesque turpis nibh, porttitor nec venenatis eu, pulvinar in augue. And now an ordered list: how exciting!</p>

      <ol>
        <li>dui neque eleifend lorem, a auctor libero turpis at sem.</li>
        <li>Aliquam ut porttitor urna.</li>
        <li>Nulla facilisi</li>
      </ol>
    </article>
  </div>
</section>

Nous avons un élément {{htmlelement("section")}}  avec une class  info-box, qui contient une {{htmlelement("ul")}} et une {{htmlelement("div")}}. La liste non ordonnée contient trois éléments de liste avec des liens à l'intérieur, qui deviendront les véritables onglets sur lesquels il faudra cliquer pour afficher nos panneaux de contenu. la div contient trois éléments {{htmlelement("article")}} , qui constitueront les panneaux de contenu correspondant à chaque onglet. Chaque panneau contient un échantillon de contenu.

L'idée ici est que nous allons donner aux onglets l'aspect d'un menu de navigation horizontal standard, et que nous allons donner aux panneaux l'aspect d'être superposés en utilisant un positionnement absolu. Nous vous donnerons également un peu de JavaScript à inclure dans votre page pour afficher le panneau correspondant lorsqu'un onglet est pressé, et nous donnerons un style à l'onglet lui-même. Vous n'aurez pas besoin de comprendre le JavaScript lui-même à ce stade, mais vous devriez penser à apprendre quelques bases de JavaScript dès que possible - plus les fonctionnalités de votre interface utilisateur deviennent complexes, plus il est probable que vous aurez besoin de JavaScript pour implémenter les fonctionnalités souhaitées.

Configuration générale

Pour commencer, ajoutez ce qui suit entre vos balises {{HTMLElement("style")}} d’ouverture et de fermeture :

html {
  font-family: sans-serif;
}

* {
  box-sizing: border-box;
}

body {
  margin: 0;
}

Ceci est juste une configuration générale pour définir une police sans empattement sur notre page, utiliser le modèle border-box {{cssxref("box-sizing")}}, et se débarrasser de lamarge de {{htmlelement("body")}} par défaut.

Ensuite, ajoutez ce qui suit juste en dessous de votre CSS précédent:

.info-box {
  width: 450px;
  height: 400px;
  margin: 0 auto;
}

Cela définit une largeur et une hauteur spécifiques sur le contenu, et le centre sur l’écran en utilisant l’ancienne astuce margin: 0 auto. Précédemment dans le cours, nous avons déconseillé de définir une hauteur fixe sur les conteneurs de contenu si possible; c’est ok dans cette circonstance parce que nous avons du contenu fixe dans nos onglets. Il semble également un peu choquant d’avoir différents onglets à différentes hauteurs.

2. Styliser nos onglets

Maintenant, nous voulons styliser les onglets pour qu’ils ressemblent à des onglets - fondamentalement, il s’agit d’un menu de navigation horizontal, mais au lieu de charger différentes pages Web lorsqu’elles sont cliquées comme nous l’avons vu précédemment dans le cours, elles provoquent l’affichage de différents panneaux sur la même page. Tout d’abord, ajoutez la règle suivante en bas de votre CSS pour supprimer le {{cssxref("padding-left")}} et {{cssxref("margin-top")}} par défaut de la liste non ordonnée :

.info-box ul {
  padding-left: 0;
  margin-top: 0;
}

Note: Nous utilisons des sélecteurs descendants avec .info-box au début de la chaîne tout au long de cet exemple — ceci afin que nous puissions insérer cette fonctionnalité dans une page avec d’autres contenus déjà dessus, sans craindre d’interférer avec les styles appliqués à d’autres parties de la page.

Ensuite, nous allons styliser les onglets horizontaux - les éléments de liste sont tous flottants à gauche pour les faire s’asseoir dans une ligne ensemble, leur {{cssxref("list-style-type")}} est défini sur none pour se débarrasser des puces, et leur {{cssxref("width")}} est définie à 150px 150px afin qu’ils s’adaptent confortablement à travers l’infobox. Les {{htmlelement("a")}} sont définis pour {{cssxref("display")}} inline-block afin qu’ils rentrent dans une ligne tout en restant stylisable,et ils sont stylisés de manière appropriée pour les boutons de tabulation, à l’aide d’une variété d’autres propriétés.

Ajoutez le CSS suivant :

.info-box li {
  float: left;
  list-style-type: none;
  width: 150px;
}

.info-box li a {
  display: inline-block;
  text-decoration: none;
  width: 100%;
  line-height: 3;
  background-color: red;
  color: black;
  text-align: center;
}

Enfin, pour cette section, nous allons définir quelques styles sur les états de lien. Tout d’abord, nous allons définir les états :focus et :hover des onglets pour qu’ils soient différents lorsqu’ils sont focalisés/survolés, offrant aux utilisateurs un retour visuel. Deuxièmement, nous allons définir une règle qui met le même style sur l’un des onglets lorsqu’une class est active est présente dessus. Nous définirons cela à l’aide de JavaScript lorsque vous cliquerez sur un onglet. Placez le CSS suivant sous vos autres styles :

.info-box li a:focus, .info-box li a:hover {
  background-color: #a60000;
  color: white;
}

.info-box li a.active {
  background-color: #a60000;
  color: white;
}

3. Stylisation des panneaux

Le prochain travail consiste à styliser nos panneaux. Allons-y!

Tout d’abord, ajoutez la règle suivante pour styliser le conteneur .panels {{htmlelement("div")}} . Ici, nous définissons simplement une {{cssxref("height")}} fixe pour nous assurer que les panneaux s’adaptent parfaitement à l’intérieur de l'info-box, {{cssxref("position")}} relative pour définir le {{htmlelement("div")}} comme contexte de positionnement, afin que vous puissiez ensuite placer les éléments enfants positionnés par rapport à elle et non par rapport à l'élement {{htmlelement("html")}}, et enfin nous {{cssxref("clear")}} le float défini dans le CSS ci-dessus afin qu’il n’interfère pas avec le reste de la mise en page..

.info-box .panels {
  height: 352px;
  position: relative;
  clear: both;
}

Finally for this section, we will style the individual {{htmlelement("article")}} elements that comprise our panels. The first rule we'll add will absolutely {{cssxref("position")}} the panels, and make them all sit flush to the {{cssxref("top")}} and {{cssxref("left")}} of their {{htmlelement("div")}} container — this part is absolutely key to this whole layout feature, as it makes the panels sit on top of one another. The rule also gives the panels the same set height as the container, and gives the content some padding, a text {{cssxref("color")}}, and a {{cssxref("background-color")}}.

The second rule we'll add here makes it so that a panel with a class of active-panel set on it will have a {{cssxref("z-index")}} of 1 applied to it, which will make it sit above the other panels (positioned elements have a z-index of 0 by default, which would put them below). Again, we'll add this class using JavaScript at the appropriate time.

.info-box article {
  position: absolute;
  top: 0;
  left: 0;
  height: 352px;
  padding: 10px;
  color: white;
  background-color: #a60000;
}

.info-box .active-panel {
  z-index: 1;
}

Ajout de notre JavaScript

La dernière étape pour faire fonctionner cette fonctionnalité consiste à ajouter du code JavaScript. Placez le bloc de code suivant, exactement comme écrit entre votre balise d'ouverture et de fermeture {{htmlelement("script")}} (vous les trouverez en fin du contenu HTML):

var tabs = document.querySelectorAll('.info-box li a');
var panels = document.querySelectorAll('.info-box article');

for(i = 0; i < tabs.length; i++) {
  var tab = tabs[i];
  setTabHandler(tab, i);
}

function setTabHandler(tab, tabPos) {
  tab.onclick = function() {
    for(i = 0; i < tabs.length; i++) {
      tabs[i].className = '';
    }

    tab.className = 'active';

    for(i = 0; i < panels.length; i++) {
      panels[i].className = '';
    }

    panels[tabPos].className = 'active-panel';
  }
}

Ce code effectue les opérations suivantes :

  • 1. Tout d’abord, nous enregistrons une référence à tous les onglets et à tous les panneaux dans deux variables tabs and panels, afin que nous puissions facilement les modifier plus tard.
  • 2. Ensuite, nous utilisons une boucle for pour parcourir tous les onglets et exécuter une fonction appelée setTabHandler() sur chacun d’eux, qui configure les fonctionnalités qui doivent se produire lorsque l’on clique sur chacun d’eux. Lors de l’exécution, la fonction reçoit une référence à l’onglet particulier pour lequel elle est exécutée et un numéro d’index i qui identifie la position de l’onglet dans le tabs array.
  • 3. Dans la fonction setTabHandler(), l’onglet a un gestionnaire d’événements onclick défini sur elle, de sorte que lorsque l’utilisateur clique sur l’onglet, les événements suivants se produisent :
    • Une boucle for est utilisée pour parcourir tous les onglets et supprimer toutes les classes qui y sont présentes.
    • Une class active est définie sur l’onglet sur lequel vous avez cliqué - rappelez-vous que cette classe a une règle associée dans le CSS qui définit la même couleur {{cssxref("color")}} et {{cssxref("background-color")}} sur l’onglet que pour les panneaux.
    • Une boucle for est utilisée pour parcourir tous les panneaux et supprimer toutes les classes qui y sont présentes.
    • Une classe de active-panel est définie sur le panneau qui correspond à l’onglet sur lequel vous avez cliqué — rappelez-vous précédemment que cette classe a une règle associée dans le CSS qui définit son {{cssxref("z-index")}} à 1, ce qui le fait apparaître au-dessus des autres panneaux.

C’est tout pour le premier exemple. Gardez votre code ouvert, car nous l’ajouterons dans le second.

Un panneau caché coulissant

Le dernier exemple que nous présenterons ici est un panneau qui glisse sur et en dehors de l’écran en appuyant sur une icône - comme mentionné précédemment, c’est populaire pour des situations telles que les mises en page mobiles, où les espaces d’écran disponibles sont petits, vous ne voulez donc pas en utiliser la majeure partie en affichant un menu ou un panneau d’informations au lieu du contenu utile.

Notre exemple fini ressemblera à ceci:

Note: Vous pouvez voir l’exemple terminé en live à hidden-info-panel.html (source code). Check it out to get an idea of what you will be building in this section of the article.

Comme point de départ, faites une copie locale de hidden-info-panel-start.html à partir de notre dépôt Github. Cela ne fait pas suite à l’exemple précédent, donc un nouveau démarrage est requis. Jetons un coup d’œil au code HTML dans le fichier:

<label for="toggle">❔</label>
<input type="checkbox" id="toggle">
<aside>

  ...

</aside>

Pour commencer ici, nous avons un element {{htmlelement("label")}} et un élement {{htmlelement("input")}} — <label> sont normalement utilisés pour associer une étiquette de texte à un élément de formulaire à des fins d’accessibilité (permettant à un utilisateur d’écran de voir quelle description va avec quel élément de formulaire). Ici, il est associé à la checkbox <input> en utilisant les attributs for et id attributes.

Note: Nous avons mis un point d’interrogation spécial dans notre code HTML pour agir comme notre icône d’information - cela représente le bouton qui sera enfoncé pour afficher / masquer le panneau.

Ici, nous allons utiliser ces éléments dans un but légèrement différent - un autre effet secondaire utile de l'élement <label> est que vous pouvez cliquer sur l’étiquette d’une case à cocher pour vérifier la case à cocher, ainsi que sur la case à cocher elle-même. Cela a conduit au h ack de case à cocher bien connu, qui fournit un moyen sans JavaScript de contrôler un élément en découlant un bouton. L’élément que nous allons contrôler est l'élément {{htmlelement("aside")}} qui suit les deux autres (nous avons laissé son contenu hors de la liste de code ci-dessus pour plus de concision).

Dans les sections ci-dessous, nous expliquerons comment tout cela fonctionne.

II. Application d’un style aux éléments de formulaire

Tout d’abord, traitons les éléments de formulaire - ajoutez le CSS suivant entre vos balises {{htmlelement("style")}} :

label[for="toggle"] {
  font-size: 3rem;
  position: absolute;
  top: 4px;
  right: 5px;
  z-index: 1;
  cursor: pointer;
}

input[type="checkbox"] {
  position: absolute;
  top: -100px;
}

La première règle s’applique au <label>; nous avons ici :

  • Set a large {{cssxref("font-size")}} to make the icon nice and big.
  • Set {{cssxref("position")}} absolute on it, and used {{cssxref("top")}} and {{cssxref("right")}} to position it nicely in the top-right corner.
  • Set a {{cssxref("z-index")}} of 1 on it — this is so that when the info panel is styled and shown, it doesn't cover up the icon; instead the icon will sit on top of it so it can be pressed again to hide the info pane.
  • Used the {{cssxref("cursor")}} property to change the mouse cursor when it is hovering over the icon to a hand pointer (like the one you see when links are hovered over), as an extra visual clue to users that the icon does something interesting.

The second rule sets {{cssxref("position")}} absolute on the actual checkbox <input> element, and hides it off the top of the screen. We don't actually want to see this on our UI.

Styling the panel

Now it's time to style the actual sliding panel itself. Add the following rule to the bottom of your CSS:

aside {
  background-color: #a60000;
  color: white;

  width: 340px;
  height: 100%;
  padding: 0 20px;

  position: fixed;
  top: 0;
  right: -370px;

  transition: 0.6s all;
}

There's a lot going on here — let's discuss it bit by bit:

  • First, we set some simple {{cssxref("background-color")}} and {{cssxref("color")}} on the info box.
  • Next, we set a fixed {{cssxref("width")}} on the panel, and make its {{cssxref("height")}} the entire height of the browser viewport.
  • We also include some horizontal {{cssxref("padding")}} to space it out a bit.
  • Next we set {{cssxref("position")}}: fixed; on the panel so it will always appear in the same place, even if the page has content to scroll. We glue it to the {{cssxref("top")}} of the viewport, and set it so that by default it is offscreen to the {{cssxref("right")}}.
  • Finally, we set a {{cssxref("transition")}} on the element. Transitions are an interesting feature that allow you to make changes between states happen smoothly, rather than just going "on", "off" abruptly. In this case we are intending to make the panel slide smoothly onscreen when the checkbox is checked. (Or to put it another way, when the question mark icon is clicked — remember, clicking the <label> will check the associated checkbox! We told you it was a hack.) You will learn a lot more about...

Setting the checked state

There is one final bit of CSS to add — put the following at the bottom of your CSS:

input[type=checkbox]:checked + aside {
  right: 0px;
}

The selector is pretty complex here — we are selecting the <aside> element adjacent to the <input> element, but only when it is checked (note the use of the {{cssxref(":checked")}} pseudo-class to achieve this). When this is the case, we are setting the {{cssxref("right")}} property of the <aside> to 0px, which causes the panel to appear on the screen again (smoothly due to the transition). Clicking the label again unchecks the checkbox, which hides the panel again.

So there you have it — a rather clever JavaScript-free way to create a toggling button effect. This will work in IE9 and above (the smooth transition will work in IE10 and above.) This effect does have some concerns — this is a bit of an abuse of form elements, as they weren't intended for this purpose. In addition, the effect is not great in terms of accessibility; the label is not focusable by default, and the non-semantic use of the form elements could cause issues with screen readers. JavaScript and a link or button might be more appropriate, but it is still fun to experiment with.
